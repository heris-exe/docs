## Overview

Fieldgrid is an operations platform for **service teams**—helping you plan work, keep technicians aligned, and connect time, materials, and invoicing.

This documentation is the source of truth for:

- **Product workflows**: how admins, coordinators, and technicians use Fieldgrid day to day.
- **APIs and integrations**: how to connect external systems, automate workflows, and sync data.
- **Internal architecture and operations**: how Fieldgrid is built and how we keep it running in production.

If you are new to Fieldgrid, start with:

- **Getting started** – a fast orientation to the app.
- **Admin getting started** or **Technician getting started** – role‑specific first‑week playbooks.

If you are building an integration, jump to the **API reference**.

## How the docs are organized

The sidebar is split into three main areas:

- **Product**: concepts and guides for using Fieldgrid.
- **API reference**: endpoints, schemas, and examples for integrating with Fieldgrid.
- **Internal**: architecture, data flows, and on-call runbooks for the Fieldgrid team.

Within the **Product** tab, you will find:

- **Getting started**: how to get access, create your first customer/project/job, and invite your team.
- **Fieldgrid concepts**: core building blocks like customers, projects, jobs, work orders, jobsites, time, and inventory.
- **User guides**: end-to-end workflows, such as the admin job lifecycle, technician daily workflow, and invoicing.

## Who this documentation is for

- **Customers and partners**: use the Product and API sections to understand how to get value from Fieldgrid and how to integrate with it.
- **Fieldgrid engineers and internal teams**: use the Internal section for architecture, domain knowledge, and operational procedures.

As we evolve Fieldgrid, these docs will remain the canonical place to understand how the system works and how to work with it.
