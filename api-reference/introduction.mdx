## Base URL

All examples in this section assume the following base URL:

```bash
https://api.fieldgrid.com
```

Replace this with the appropriate environment base URL for staging or internal testing as needed.

## Authentication

The Fieldgrid API uses secure authentication (for example, bearer tokens or API keys).

```bash
curl https://api.fieldgrid.com/api/fields \
  -H "Authorization: Bearer YOUR_TOKEN"
```

In your Mintlify demo, you can highlight where tokens or keys are created in the app and how they are passed in requests.

## Error handling

The API returns standard HTTP status codes and error payloads. Focus on:

- 2xx – success.
- 4xx – client-side issues (validation, auth).
- 5xx – server-side issues.

## Key resources

The core entities covered in this API reference are:

- **Fields** – spatial areas where work and data are scoped.
- **Tasks** – units of work linked to fields.
- **Sensors** – data sources attached to fields or organizations.

Each has a dedicated page with example endpoints, shown in the sidebar.
