## Overview

**Alex** is Fieldgrid’s AI assistant. It helps users:

- Ask questions about jobs, documents, and procedures.
- Generate summaries and insights.
- Navigate context-aware suggestions from within the app.

Alex surfaces in both admin and technician workflows via:

- Dedicated AI pages.
- Modals and floating action buttons (FABs).

## Entry points

- AI pages:
  - `/ai` or similar route for `AIAssistantPage` (`fieldgrid-react/src/pages/AIAssistantPage.tsx`).
  - `/ai/chat/:id` or `AIChatPage` (`src/pages/AIChatPage.tsx`) for session-specific chats.
- In-context entry points:
  - Alex FAB in `AppLayout` via `AlexProjectBuilderFab.tsx`.
  - `AskAlexModal.tsx` for job- or project-specific questions.

- Types:

  - `src/api/ai/ai.ts`

- `/api/v1/ai/chat/quick`
- `/api/v1/ai/chat/sessions`
- `/api/v1/ai/documents`

## Core workflows

### 1. Ask Alex from a job or project

- From a job, project, or schedule view:
  - Click the **Ask Alex** button or FAB.
- The `AskAlexModal`:
  - Captures:
    - The user’s question.
    - Context (job ID, project ID).
    - Optionally related documents.
  - Sends a request to AI endpoints via `api/ai/*`.
  - Streams back responses using `streaming` APIs where supported.

### 2. Use the AI console

- On the dedicated AI page:
  - Users can:
    - Start new chat sessions.
    - Resume past sessions.
    - Search previous interactions.
  - The console uses:
    - `chatHistoryStore` for local state.
    - AI APIs for persistence and retrieval.

### 3. Upload documents for AI context

- From AI pages or admin resources:
  - Upload documents to be used as AI context.
- Backend support (as seen in `fieldgrid-fastapi` docs):
  - `/api/v1/ai/documents` for document upload.
  - Chat endpoints with optional document context parameters.

### 4. Time insights and operational help

- Components like `AlexTimeInsightsCard.tsx` use AI to:
  - Summarize time tracking data.
  - Provide recommendations or insights for admins.
