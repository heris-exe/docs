## Overview

Jobs and work orders are at the heart of Fieldgrid’s operations model. This guide walks through the **admin job lifecycle**:

- Creating jobs under projects.
- Breaking work into work orders.
- Assigning technicians.
- Tracking progress and closing out work.

On the web, this flow is centered around the `/admin/jobs` routes and the generic `/jobs/:id` router in `fieldgrid-react`.

## Entry points

- From the admin sidebar:
  - `/admin/jobs` → job management list.
- From projects or customers:
  - Links to `/admin/jobs` filtered by project or customer.
- Direct job links:
  - `/jobs/:id` via `JobRouter` (`src/pages/JobRouter.tsx`).

## Lifecycle steps

### 1. Create or import jobs

- From `/admin/jobs`:
  - Click **Create job** → opens `JobCreateModal`.
  - Provide:
    - Project or customer.
    - Job name and description.
    - Dates, status, and optional budget-related fields.
    - Lead technician (where applicable).
- Optionally:
  - Use `JobImportModal` to import multiple jobs from a structured source.
- The frontend calls job create endpoints via `useJobs` mutations and `api/jobs/jobs.ts`.

### 2. Define work orders

- In `JobDetailsPage` → **Work orders** tab:
  - Create work orders linked to the job.
  - Each work order:
    - Has a title and description.
    - Is assigned to one or more technicians.
    - Has priority, due dates, and status fields.
- Subtasks:
  - Work orders can be further broken down into subtasks using `SubTaskCreateModal` and `src/api/work-orders/sub-tasks.ts`.

### 3. Assign technicians and schedule work

- Assign technicians to jobs and/or work orders.
- Use scheduling features:
  - In the **Schedule** area (`/admin/schedule`) for a calendar or board view.
  - Directly on the job or work order details.
- Scheduling data is modeled in:
  - `src/types/jobs/schedule.ts`
  - APIs under `src/api/jobs/schedules.ts`.

### 4. Track progress and collaboration

- Use the **Job details** view to track:
  - Work order statuses.
  - Technician assignments.
  - Time entries and checklists.
  - Comments, site logs, and files.
- Activity and audit history:
  - Fetched via `useActivities.ts` and APIs in `src/api/activities/*`.
  - Exposed in UI components like activity logs and audit trails.

### 5. Close out work

- When a job is completed:
  - All work orders should be completed or canceled.
  - Relevant checklists and logs should be finalized.
  - Time entries and inventory transactions should be reconciled.
- The job’s status is updated via job update APIs.
- Downstream flows:
  - Generating or updating invoices.
  - Analytics and reporting.
