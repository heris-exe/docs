## Overview

Super admins manage the **Fieldgrid platform itself**, not just a single organization. They are responsible for:

- Managing organizations and their billing plans.
- Controlling access to demo environments.
- Overseeing global audit logs and announcements.

This guide explains the super admin surfaces under `/superadmin` and how demo access fits into the overall experience.

## Entry points

- Super admin namespace:
  - `/superadmin/dashboard`
  - `/superadmin/companies`
  - `/superadmin/plans-billing`
  - `/superadmin/users`
  - `/superadmin/demo-access`
  - `/superadmin/audit-logs`
  - `/superadmin/announcements`
  - `/superadmin/settings`
  - `/superadmin/help`

Access is limited to users with the super admin role; route-level checks are enforced via:

- `canAccessRoute` and route guards in `src/utils/routing.ts` and `src/App.tsx`.

## Super admin workflows

### 1. Manage organizations and billing plans

From `/superadmin/companies` and `/superadmin/plans-billing`:

- View the list of organizations.
- Inspect:
  - Current plan.
  - Seat counts and active modules.
  - Trial status and important billing flags.
- Use `SuperAdminCompanyBillingPanel` and related components to:
  - Adjust plans and seat counts.
  - Trigger manual subscription changes (with care).
  - View billing history at a high level.

### 2. Manage users and access

From `/superadmin/users`:

- View all users across organizations.
- Filter and search by:
  - Email, role, or organization.
- Perform privileged actions (as allowed by policy):
  - Promote or demote roles (e.g., grant or revoke super admin).
  - Lock or unlock accounts.

### 3. Demo access workflows

From `/superadmin/demo-access`:

- Create and manage **demo environments** and access tokens.
- Typical tasks:
  - Generate demo invites or access links.
  - Reset demo environments.
  - View active demo sessions.
- The frontend uses `demoAccess` APIs to coordinate with backend demo-management logic.

## Global oversight

### 1. Announcements

From `/superadmin/announcements`:

- Create platform-wide announcements.
- Target:
  - Specific organizations.
  - All organizations.
  - Specific roles.

### 2. Audit logs

From `/superadmin/audit-logs`:

- View global audit events across organizations.
- Filter by:
  - Organization.
  - User.
  - Entity type or action type.

These views are crucial for:

- Security investigations.
- Debugging multi-tenant issues.
- Ensuring compliance and governance.
