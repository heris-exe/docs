## When to use this guide

Use this guide when a new customer or internal farm needs to be set up in Fieldgrid for the first time.

## Step 1 – Create the organization (if needed)

- Ensure the organization exists in Fieldgrid.
- Configure basic settings such as default units, time zone, and branding.

## Step 2 – Model fields

1. Work with the customer to identify which physical areas should become fields.
2. In Fieldgrid, create a field for each area, capturing:
   - Name and optional code.
   - Location (drawn on a map or entered as coordinates).
   - Crop and other relevant metadata.

## Step 3 – Configure roles and invite users

1. Define which roles are needed (field techs, managers, admins).
2. Invite users via **Settings → Team**.
3. Assign roles in line with the customer’s responsibilities and security requirements.

## Step 4 – Connect sensors (if applicable)

1. Identify which sensors are installed or planned.
2. For each sensor:
   - Create it in Fieldgrid.
   - Associate it with the relevant field(s).
   - Verify that data is flowing and visible in dashboards.

## Step 5 – Run a pilot workflow

1. Create a small set of tasks in the new fields (for example, a scouting task).
2. Assign tasks to field techs and set clear due dates.
3. Have assignees complete the tasks and log data from the field.

## Step 6 – Review with the customer

- Walk through dashboards and reports to show captured data.
- Adjust configuration (fields, roles, notifications) based on feedback.
