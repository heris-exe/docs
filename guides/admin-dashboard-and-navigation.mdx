## Overview

The **admin dashboard** is the primary control center for operations leaders. From here admins can:

- See an overview of active work.
- Jump into jobs, projects, resources, analytics, and billing.
- Navigate across all admin surfaces using the left sidebar and top header.

This experience is built around the `/admin` route namespace and the shared `AppLayout` shell in `fieldgrid-react`.

## Entry points

- After authentication and onboarding, admins are routed to:
  - `/admin/dashboard`
- Subsequent navigations rely on the sidebar and route-based links within pages.

Routing and role-based redirection are implemented in:

- `fieldgrid-react/src/App.tsx` – `ProtectedRoute` and top-level `<Routes>`.
- `fieldgrid-react/src/utils/routing.ts` – `getDashboardRoute`, `canAccessRoute`.

## Layout shell

The admin experience shares a common layout defined in:

- `fieldgrid-react/src/layouts/AppLayout.tsx`

The layout provides:

- **Sidebar** with sectioned navigation (Dashboard, Jobs, Projects, Resources, Inventory, Invoices, Time, Customers, Analytics, Audit Log, Settings, Integrations, Help).
- **Header** with:
  - Organization selector.
  - Global search.
  - Notifications and announcements.
  - User menu and Alex (AI assistant) entry points.
- **Content area** controlled by React Router’s `<Outlet>`.

The layout consumes state and hooks such as:

- `useAuthStore` – current user and role.
- `useOrganization` – active organization context.
- `useTaskLabel`, `useBreadcrumbs` – contextual UI.
- `useWebSocketNotifications` – live notifications.

## Core admin routes

Under the `/admin` namespace, key pages include:

- `/admin/dashboard` → `AdminDashboard` (`src/pages/AdminDashboard.tsx`)
- `/admin/jobs` → job management.
- `/admin/projects` → project list and details.
- `/admin/resources` → document and file management.
- `/admin/inventory` → inventory management.
- `/admin/invoices` → invoicing.
- `/admin/expenses` → expenses and approvals.
- `/admin/time-entries` → time tracking oversight.
- `/admin/schedule` → scheduling.
- `/admin/customers` → customer management.
- `/admin/team` → team management.
- `/admin/analytics` → reporting and analytics.
- `/admin/audit-log` → audit history.
- `/admin/settings` → organization and app settings.
- `/admin/integrations` → integrations configuration.
- `/admin/help` → in-app help and support.

Each of these pages is implemented in `fieldgrid-react/src/pages` as a dedicated React component that uses:

- Domain-specific React Query hooks from `src/hooks`.
- Domain-specific API modules from `src/api`.
- Shared UI primitives from `src/components/ui`.

## Navigation patterns

- **Sidebar navigation**
  - Driven by a configuration of nav items tied to routes and permissions.
  - Items may be conditionally shown based on role, entitlements, or feature flags.
- **Breadcrumbs**
  - Derived from the current route and optional route-level metadata.
  - Implemented via `useBreadcrumbs` and rendered in `AppLayout`.
- **Command palette / search**
  - Provides quick navigation and global search across jobs, projects, and other entities.
- **Cross-role navigation**
  - Some routes (e.g., `/jobs/:id`) are shared across roles using routers like `JobRouter`.
  - `canAccessRoute` ensures that technicians cannot access admin-only pages.
